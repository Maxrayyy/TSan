# 宿松拖三 - 产品需求规格文档 (PRD)

> 版本: v1.0 | 日期: 2026-04-07 | 状态: 初稿

## 1. 产品概述

### 1.1 产品愿景
开发一款基于 Web 的多人在线"宿松拖三"扑克游戏，支持好友邀请、房间制对战，还原宿松地区经典拖三玩法。

### 1.2 目标用户
- 宿松及安庆地区扑克爱好者
- 离乡在外想玩家乡牌的用户
- 任何对地方扑克感兴趣的玩家

### 1.3 产品定位
- 轻量级 Web 应用，无需下载安装
- 手机端优先（响应式设计），同时支持 PC 浏览器
- 好友私房间为核心场景（非匹配制）

### 1.4 版本规划

| 版本 | 范围 | 说明 |
|------|------|------|
| **MVP (v0.1)** | 核心牌局 | 一副牌版、4人房间、基础出牌/比大小/计分 |
| **v0.2** | 社交功能 | 好友系统、邀请链接、房间聊天 |
| **v0.3** | 体验优化 | 动画效果、音效、断线重连、观战 |
| **v1.0** | 完整版 | 扑牌声明、三副牌版、积分排行榜 |

## 2. 功能需求

### 2.1 用户系统

#### F-001 注册与登录
- 支持手机号/邮箱注册
- 支持微信OAuth登录（可选，v0.2）
- 支持游客模式：输入昵称即可快速进入（MVP必须）
- 用户信息：昵称、头像（默认头像集）、积分

#### F-002 用户资料
- 查看/编辑昵称、头像
- 查看历史战绩统计（胜率、总局数、最高连胜）

### 2.2 房间系统

#### F-010 创建房间
- 房主创建房间，生成6位房间号
- 房主可设置：
  - 游戏版本：一副牌 / 三副牌
  - 目标分数：自定义（默认2000分）
  - 是否允许观战
- 创建后进入等待大厅

#### F-011 加入房间
- 通过房间号加入
- 通过邀请链接加入（链接含房间号，打开即进入）
- 通过好友邀请加入（v0.2）

#### F-012 房间大厅
- 显示4个座位，已入座玩家显示头像和昵称
- 玩家可选择座位（对面为队友）
- 房主可踢人、开始游戏（需4人满座）
- 实时显示玩家进出状态
- 房间内文字聊天（快捷短语 + 自由输入）

#### F-013 邀请机制
- 生成邀请链接（含房间号），可一键复制/分享
- 链接有效期：房间存在期间有效
- 支持生成邀请二维码（v0.2）

### 2.3 核心游戏流程

#### F-020 发牌阶段
- 一副牌（52张，无大小王），每人13张
- 发牌动画：逐张发到各玩家位置
- 发牌完成后，玩家手牌自动排序（按点数从小到大）

#### F-021 确定先手
- 第一局：各抽一张牌，最大者先出
- 后续局：上局末游先出（或按规则确定）

#### F-022 扑牌声明（v1.0）
- 发牌后，出牌前，玩家可选择声明类型
- 声明类型：黑的/报的/小摊/大摊/定摊
- 限时10秒做出选择，超时默认"黑的"

#### F-023 出牌阶段
- 轮流出牌，顺时针方向
- 当前出牌者高亮提示，倒计时30秒
- 玩家可选择"出牌"或"不要"（pass）
- 出牌时自动检测牌型合法性
- **核心限制**：接风时不能出3或含3的牌
- 超时自动pass（非首次出牌时）；首次出牌超时自动出最小合法牌

#### F-024 牌型识别与验证
- 系统自动识别玩家选中的牌构成的牌型
- 跟牌时验证：必须同牌型且更大，或出炸弹/特殊牌型压制
- 非法出牌给出提示，不允许打出

#### F-025 回合结算
- 每回合（所有人pass后）结算本回合分数牌
- 实时更新双方队伍得分
- 显示本回合赢家动画

#### F-026 拖三与憋三
- 拖三判定：最后一手为纯3且无人接，触发拖三动画与加分
- 憋三判定：游戏结束时手中有3，触发憋三动画与扣分
- 实时显示拖三/憋三计数

#### F-027 游戏结束
- 触发条件：双带花 或 3人出完牌
- 结算页面显示：
  - 各玩家排名（头游/二游/三游/末游）
  - 本局得分明细（抓分 + 拖三 - 憋三 + 名次分）
  - 累计总分
  - 拖三/憋三详情
- "再来一局" / "返回房间" 按钮

### 2.4 实时通信

#### F-030 游戏状态同步
- 所有游戏操作实时同步到4位玩家
- 延迟容忍：<200ms（正常网络环境下）
- 对手手牌显示牌背，仅显示剩余张数

#### F-031 断线重连
- 断线后30秒内可自动重连
- 重连后恢复完整游戏状态
- 断线期间自动pass（非出牌方）或延长超时（出牌方等待60秒）
- 超过60秒未重连，由AI托管（v0.3）

#### F-032 观战模式（v0.3）
- 房间设置允许观战时，第5人及以后进入为观众
- 观众可看到所有玩家手牌（延迟1回合显示，防作弊）
- 观众不可发言影响牌局

### 2.5 社交功能（v0.2）

#### F-040 好友系统
- 添加好友（通过ID/昵称搜索）
- 好友列表（在线/离线状态）
- 邀请好友进入房间

#### F-041 游戏内交互
- 快捷表情/文字：预设短语（"快点啊"、"打得好"等）
- 自由文字聊天
- 表情包（v0.3）

## 3. 非功能需求

### 3.1 性能
- 页面首次加载 < 3秒（3G网络）
- 游戏操作响应 < 200ms
- 支持同时在线 1000 个房间（初期目标）

### 3.2 可用性
- 移动端优先的响应式设计
- 最小支持屏幕：320px 宽
- 支持竖屏和横屏
- 触屏操作：点击选牌，上滑/按钮出牌

### 3.3 兼容性
- 浏览器：Chrome 80+, Safari 14+, Firefox 80+, 微信内置浏览器
- 设备：iOS 13+, Android 8+, Windows/Mac 现代浏览器

### 3.4 安全
- 所有牌局逻辑在服务端执行，客户端仅做展示
- 防止篡改出牌数据
- 房间号防暴力枚举（限流 + 验证码）
- 通信全程 HTTPS/WSS

## 4. UI/UX 要求

### 4.1 整体风格
- 中国风/棋牌风格，色调以绿色牌桌为主
- 扑克牌使用标准样式，清晰可辨
- 简洁直观，减少学习成本

### 4.2 关键界面

| 页面 | 核心元素 |
|------|----------|
| **首页** | Logo、快速开始（游客）、登录/注册、创建房间、加入房间 |
| **房间大厅** | 4个座位、房间号、邀请按钮、聊天区、准备/开始按钮 |
| **游戏主界面** | 牌桌（4方位）、自己手牌（底部扇形展开）、出牌区域、得分面板、操作按钮（出牌/不要/提示）、倒计时 |
| **结算页面** | 排名列表、得分明细、拖三憋三详情、操作按钮 |

### 4.3 手牌交互
- 点击选中/取消选中（高亮上移）
- 可选中多张后点"出牌"
- "提示"按钮：自动推荐可出的牌组合
- 手牌自动排序：按点数排列，3在最左

### 4.4 信息展示
- 自己手牌：底部完整展示
- 对手手牌：牌背 + 剩余张数
- 队友标识：特殊颜色/图标
- 当前出牌者：高亮 + 倒计时环
- 场上已出的牌：中央区域展示，上一回合出的牌半透明保留

## 5. 数据与分析（v0.3+）

### 5.1 核心指标
- DAU / MAU
- 平均每日对局数
- 平均对局时长
- 房间创建→开局转化率
- 邀请链接点击→加入转化率

### 5.2 游戏数据
- 各牌型出现频率
- 拖三/憋三成功率
- 双带花发生率
- 平均每局回合数

## 6. 约束与假设

### 6.1 约束
- MVP阶段仅实现一副牌版
- 不做匹配系统（仅房间制）
- 不做充值/付费功能
- 服务端权威：所有游戏逻辑由服务端裁决

### 6.2 假设
- 用户具备基本扑克游戏经验
- 用户处于稳定的网络环境中（WiFi/4G）
- 初期用户通过微信等社交渠道传播
