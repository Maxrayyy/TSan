# 宿松拖三 - 任务拆分与开发计划

> 版本: v1.0 | 日期: 2026-04-07

## 总体原则

- 采用垂直切片开发：每个阶段交付可运行的功能
- 优先实现核心游戏循环，后续迭代增强体验
- 测试驱动：核心游戏逻辑必须有单元测试

## 阶段三：核心游戏逻辑 (Phase 2) ⭐最关键阶段

### P2-1 牌堆与发牌
- [ ] 实现 `Card` 类
- [ ] 实现 `Deck` 类（生成52张牌、Fisher-Yates洗牌、发牌）
- [ ] 编写发牌测试（每人13张、无遗漏无重复）
- **产出**: 可洗牌发牌的牌堆模块

### P2-2 牌型检测
- [ ] 实现 `HandDetector`
  - [ ] 单牌、对子、三张、炸弹
  - [ ] 顺子（>=5张，不含2）
  - [ ] 连对（>=2对连续，不含2）
  - [ ] 连三（>=2组连续，不含2）
  - [ ] 三带两
  - [ ] 高高（两组三带两）
  - [ ] 510K / 纯510K
  - [ ] 通天龙 / 纯色通天龙
  - [ ] 纯龙（同花顺子）
- [ ] 编写每种牌型的单元测试（正例+反例）
- **产出**: 输入一组牌 → 输出牌型或null

### P2-3 牌型比较
- [ ] 实现 `HandComparator`
  - [ ] 同类型比较（rank比较）
  - [ ] 跨类型压制（510K > 普通，炸弹 > 510K，等）
  - [ ] 纯龙 vs 连对的特殊规则
  - [ ] 通天龙、纯色通天龙的等级
- [ ] 编写比较测试（覆盖压制关系表中所有情况）
- **产出**: 输入两个牌型 → 判断谁大/不可比较

### P2-4 出牌验证
- [ ] 实现 `PlayValidator`
  - [ ] 轮次检查（是否轮到该玩家）
  - [ ] 持牌检查（玩家是否有这些牌）
  - [ ] 牌型合法性
  - [ ] 接风限制（不能出含3的牌）
  - [ ] 跟牌合法性（必须压过上家）
- [ ] 编写验证测试
- **产出**: 完整的出牌合法性校验

### P2-5 拖三与憋三
- [ ] 实现 `TuoSanRule`
  - [ ] 拖三判定（纯3 + 无人接）
  - [ ] 憋三判定（游戏结束手中有3）
  - [ ] 4张3炸弹豁免
- [ ] 编写拖三/憋三测试
- **产出**: 准确判定拖三与憋三

### P2-6 计分系统
- [ ] 实现 `Scorer`
  - [ ] 单回合分数计算（5/10/K/红桃5）
  - [ ] 终局结算（排名得分 + 抓分 + 拖三 - 憋三）
  - [ ] 双带花结算
  - [ ] 末游分数归头游队伍
- [ ] 编写计分测试（各种终局场景）
- **产出**: 完整计分系统

### P2-7 游戏引擎（状态机）
- [ ] 实现 `GameEngine` 类
  - [ ] 状态机：DEALING → PLAYING → FINISHED
  - [ ] `deal()`: 初始化游戏、洗牌发牌
  - [ ] `play()`: 处理出牌（验证→从手牌移除→更新状态→检查出完→轮转）
  - [ ] `pass()`: 处理不要（计数→检查回合结束→检查拖三→轮转）
  - [ ] `getPlayerView()`: 生成客户端视图（隐藏其他人手牌）
  - [ ] 回合结束处理（分数归赢家、拖三检查、出牌权转移）
  - [ ] 游戏结束处理（双带花检测、终局结算）
  - [ ] 超时处理（自动pass/出最小牌）
- [ ] 编写完整游戏流程集成测试（模拟一局完整对局）
- **产出**: 独立可运行的游戏引擎，通过测试验证正确性

## 测试策略

| 层级 | 工具 | 覆盖范围 |
|------|------|----------|
| **单元测试** | Vitest | 游戏核心逻辑（牌型检测、比较、验证、计分） |
| **集成测试** | Vitest + SuperTest | API接口、Socket事件流 |
| **E2E测试** | Playwright（可选） | 完整对局流程 |

### 核心测试用例清单

**牌型检测：**
- 每种牌型的正确识别（13种牌型 × 多个用例）
- 非法牌型返回null（随机组合、超长顺子含2等）

**牌型比较：**
- 同类型大小比较（对子A > 对子K）
- 跨类型压制（炸弹 > 对子，510K > 单牌）
- 不可比较情况（顺子5张 vs 顺子6张 → null）

**出牌验证：**
- 正常出牌通过
- 接风出3被拒
- 出的牌不够大被拒
- 不是自己的回合被拒
- 出了自己没有的牌被拒

**拖三/憋三：**
- 出纯3且无人接 → 触发拖三
- 出含3但不全是3 → 不触发拖三
- 游戏结束手中有3 → 憋三
- 4张3炸弹 → 不算憋三

**计分：**
- 红桃5 = 55分，其他5 = 5分
- 双带花结算
- 末游分数归头游队伍

**完整对局模拟：**
- 4个虚拟玩家按预设策略打完一局，验证最终结算正确
