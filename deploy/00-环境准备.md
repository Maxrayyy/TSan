# 本地开发环境搭建

从克隆仓库到项目运行的完整指南，适用于 WSL2 / Ubuntu / Debian 等 Linux 环境。

## 前置要求

| 软件             | 最低版本 | 说明                         |
| ---------------- | -------- | ---------------------------- |
| Node.js          | >= 20    | 推荐 LTS 版本                |
| pnpm             | >= 9     | 包管理器                     |
| Docker + Compose | -        | 用于运行 PostgreSQL 和 Redis |

> 如果不想使用 Docker，也可以直接安装 PostgreSQL 16 和 Redis 7，见附录 A。

---

## 1. 安装基础工具

### Node.js

```bash
# 通过 NodeSource 安装（推荐）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 或使用 nvm
# curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
# nvm install 20
```

### pnpm

```bash
npm install -g pnpm
```

### Docker

```bash
# 方案一：Ubuntu 官方仓库（简单，推荐 WSL 用户）
sudo apt update
sudo apt install -y docker.io docker-compose-v2

# 方案二：Docker 官方仓库（版本更新）
# 参考 https://docs.docker.com/engine/install/ubuntu/

# 配置免 sudo
sudo usermod -aG docker $USER
newgrp docker    # 或重新打开终端

# 启动 Docker 服务（WSL 中需手动启动）
sudo service docker start
```

**国内网络注意：** 如果拉取 Docker 镜像超时，需要配置镜像加速器：

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<'EOF'
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me"
  ]
}
EOF
sudo service docker restart
```

> 镜像源可用性会变化，如果以上地址失效，请搜索最新可用的 Docker 镜像加速器。

## 2. 克隆与安装

```bash
git clone <仓库地址> TSan
cd TSan
pnpm install
```

## 3. 配置环境变量

```bash
cp .env.example .env
```

默认的开发环境配置无需修改即可使用：

```
DATABASE_URL="postgresql://tuosan:tuosan_dev@localhost:5432/tuosan"
REDIS_URL="redis://localhost:6379"
JWT_SECRET="your-jwt-secret-change-in-production"
JWT_REFRESH_SECRET="your-refresh-secret-change-in-production"
PORT=3000
NODE_ENV=development
```

---

## 4. 启动数据库服务

```bash
docker compose up -d
```

验证服务状态：

```bash
# 查看容器状态（应显示 postgres 和 redis 均为 Up，且 PORTS 列有端口映射）
docker compose ps

# 验证连通性
docker compose exec postgres pg_isready
docker compose exec redis redis-cli ping
```

> 如果端口被占用（如本地已安装 PostgreSQL 或 Redis），需要先停掉对应服务：
> `sudo service postgresql stop` 或 `sudo service redis-server stop`

## 常见问题

### Docker 服务未启动（WSL）

WSL 中 Docker 不会自动启动，每次打开 WSL 需手动执行：

```bash
sudo service docker start
```

### 端口被占用

```bash
# 查看占用端口的进程
lsof -i :3000   # 后端
lsof -i :5432   # PostgreSQL
lsof -i :6379   # Redis

# 终止占用进程
kill <PID>
```

### Prisma 相关问题

本项目使用 Prisma 7，配置方式与旧版不同：

- 数据库连接 URL 配置在 `packages/server/prisma.config.ts` 的 `datasource.url` 中（不在 schema.prisma 里）
- PrismaClient 通过 `@prisma/adapter-pg` 适配器连接数据库（不使用内置引擎）
- 开发环境推荐使用 `npx prisma db push` 同步 schema，而非 `prisma migrate dev`

## 附录 A：无 Docker 方案（直装 PostgreSQL + Redis）

如果不想使用 Docker，可以直接在系统中安装：

**PostgreSQL 16：**

```bash
sudo apt install -y postgresql postgresql-contrib
sudo systemctl enable postgresql
sudo systemctl start postgresql
sudo -u postgres psql -c "CREATE USER tuosan WITH PASSWORD 'tuosan_dev';"
sudo -u postgres psql -c "CREATE DATABASE tuosan OWNER tuosan;"
```

**Redis 7：**

```bash
sudo apt install -y redis-server
sudo systemctl enable redis-server
sudo systemctl start redis-server
```

安装完成后，从第 3 步（配置环境变量）继续即可。
