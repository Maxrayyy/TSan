# 阶段二完成后操作（用户与房间系统 Phase 1）

## 已完成的内容

- **P1-1** Prisma Schema（User, UserStats, GameRecord, GamePlayer, Friendship）
- **P1-2** 认证后端（游客登录、注册、登录、Token 刷新、JWT 中间件）
- **P1-3** 认证前端（首页 UI、登录注册表单、useAuthStore、API 封装）
- **P1-4** 房间后端（Redis 房间存储、房间 API、Socket.IO 事件处理）
- **P1-5** 房间前端（创建/加入房间、房间大厅 UI、Socket 连接、useRoomStore）

## 你需要做的操作

### 1. 确保 PostgreSQL 和 Redis 已启动

```bash
# Docker 方案
docker compose up -d

# 或确认本地安装的服务已运行
# PostgreSQL: 端口 5432
# Redis: 端口 6379
```

### 2. 确保 .env 文件存在

```bash
# 项目根目录
cp .env.example .env
# 编辑 .env 确认 DATABASE_URL 和 REDIS_URL 正确
```

### 3. 执行数据库迁移

```bash
cd packages/server
npx prisma migrate dev --name init
```

这会：

- 在 PostgreSQL 中创建所有表（User, UserStats, Friendship, GameRecord, GamePlayer）
- 生成 Prisma Client

如果 `prisma migrate dev` 因 Prisma 7 配置报错，尝试：

```bash
npx prisma db push
```

`db push` 会直接同步 schema 到数据库（开发环境推荐）。

### 4. 重新构建共享包

```bash
cd packages/shared
pnpm build
```

### 5. 启动后端

```bash
cd packages/server
pnpm dev
```

### 6. 启动前端

```bash
cd packages/client
pnpm dev
```

### 7. 验证功能

**认证流程：**

1. 打开 http://localhost:5173
2. 点击"快速开始" → 输入昵称 → 应成功以游客身份进入
3. 或点击"登录/注册" → 注册新账号 → 应成功登录

**房间流程：**

1. 登录后点击"创建房间" → 应跳转到房间页面
2. 复制邀请链接，在另一个浏览器窗口打开 → 应看到房间信息
3. （需要另一个账号登录才能加入）

### 8. API 测试（可选，用 curl 或 Postman）

```bash
# 游客登录
curl -X POST http://localhost:3000/api/auth/guest \
  -H "Content-Type: application/json" \
  -d '{"nickname":"测试玩家"}'

# 创建房间（用上面返回的 accessToken）
curl -X POST http://localhost:3000/api/room/create \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your_token>"

# 查询房间
curl http://localhost:3000/api/room/<roomId>
```
